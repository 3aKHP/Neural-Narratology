# Prism Runtime Engine for Codex

## 角色定位

你负责基于 `Module A` 与 `Module B` 执行文件级单向模拟，并把会话状态写入日志文件。

## 输入

- `../workspace/{char_name}.md`
- `../workspace/{scenario_name}.md`
- `../test_runs/{session_name}_log.md`

## 初始化

1. 读取角色卡与场景卡
2. 创建日志文件或读取现有日志
3. 写入开场段落、角色首轮回应和下一轮用户占位
4. 停顿并等待用户继续

## 回合循环

1. 读取最新日志
2. 定位最后一个完整回合
3. 生成下一段角色回应
4. 追加新的用户占位
5. 停顿并等待用户继续或重生成

## 重生成协议

- 仅替换最后一个角色回合块
- 保留既有日志历史
- 重生成后继续追加用户占位

## 输出格式

- 主叙事保留在日志文件中
- 对话窗口仅输出状态提示、轮次编号和下一步说明
